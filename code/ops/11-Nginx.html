# 1、Tomcat集群

集群：所谓集群，简单说就是相同的内容复制多份。

比如我们的单体应用All In One,如果一台服务器不能满足高并发的需求，可以横向扩展服务器数量。

但是服务器中的项目都是相同的。

分布式：在微服务架构中，不同服务器中的内容不完全相同。

比如某些服务器是用户模块，有些服务器部署商品或物流模块。

总结：

Tomcat集群指的是相同的Tomcat复制多份。

安装三个Tomcat(测试负载均衡) 修改主页的内容，当访问不同的Tomcat时能看到明显变化。

# 2、Nginx

项目中瓦片资源越来越多，如果提高瓦片的访问效率是一个需要解决的问题。这里，我们考虑使用Nginx来代理静态资源进行初步解决。

Nginx ("engine x") 是一个高性能的[HTTP]和[反向代理]服务器，也是一个IMAP/POP3/SMTP[服务器]。Nginx是由Igor Sysoev为[俄罗斯]访问量第二的Rambler.ru站点开发的。

何为反向代理呢？即以代理服务器来接受internet上的连接请求，然后将请求转发给内部网络上的服务器，并将从服务器上得到的结果返回给internet上请求连接的客户端，此时代理服务器对外就表现为一个服务器。

而Nginx主要完成的工作就是反向代理，比如我们向一台服务器的80端口发送了请求，该请求首先由Nginx监听到，其接受到请求内容后再转发给其他服务器，其他服务器处理完后再将结果传送给Nginx，最后由Nginx来统一返回给初始请求端。

Nginx最常见的使用方法就是作为静态资源伺服器，比如将CSS，JS，html，htm等文件进行缓存处理。

在高连接并发的情况下，Nginx是Apache服务器不错的替代品。

# 2、负载均衡

配置负载均衡 修改/etc/nginx/conf.d/目录下的default.conf文件，添加如下内容

upstream accp.site{
      server 192.168.200.150:18080 weight=1;
      server 192.168.200.150:28080 weight=2;
      server 192.168.200.150:38080 weight=1;
}

# 修改此内容，添加
# proxy_pass http://accp.site/;
location / {
    root   /usr/share/nginx/html;
    index  index.html index.htm;
    proxy_pass http://accp.site/;
}
